eggを作ってみる

Pythonのモジュールの配布方法としてeggというzipファイルを作成して配るらしい。

完全にこちらのサイトを参考にしました。ありがとうございます。mm

python開発環境の構築から作成したeggを別環境へインストールするまで pasterのインストール

$ easy_install PasteScript

雛形の作成

$ paster create -t basic_package MyPkg

これやると色々聞かれる。バージョンとか作者の名前とか。聞かれたことはsetup.py中に記録されています。

egg作成

$ cd MyPkg
$ python setup.py bdist_egg

bdist_eggってコマンドだけど、なんの略？？

とりあえず、よくわからない時の合言葉”おまじない”ってことで。 で、Mypkg/distの下にパッケージが完成！！

インストールはこれでオケ。

$ easy_install &#123;作ったeggファイル&#125;

$ python

$ pydoc sys
ヘルプが表示される

virtualenv使ってみる

独立した環境を利用するためのツールがある。virtualenvってやつで、Python3からはpyvenv(venv?)ってのを利用するらしい。

で、virtualenvを利用しやすくするものにpyenvとか、なんちゃらenvとかいろいろあるみたい。Goで書かれたdirenvってのもぐいぐい来ているらしいが、また今度。 とりあえずインストール

$ sudo pip install virtualenv

virtualenvwrapperってのを入れると環境の切り替えが楽らしい

sudo pip install virtualenvwrapper

環境設定 .bashrcに追加

#virtualenv settings
export WORKON_HOME=$HOME/.virtualenvs
. /Library/Frameworks/Python.framework/Versions/2.7/bin/virtualenvwrapper.sh

環境のファイル作る

mkvirtualenv &#123;env name&#125;

作った環境一覧

workon

とある環境をアクティブにする

workon &#123;env name&#125;

アクティブ終了

deactivate

削除するとき

rmvirtualenv &#123;env name&#125;

私の環境はMacOSXです。
